Suburban-SOC: Command Syntax & Security Update Guide

1. Overview

Following the recent security overhaul of the Suburban-SOC infrastructure, Elasticsearch authentication (X-Pack Security) has been strictly enforced. As a result, starting the ELK pipeline now requires passing a secure password via environment variables to ensure the container boot process seamlessly provisions credentials.

2. Starting the Secured ELK Stack

Secrets are supplied via scripts/setup/.env (never hard-coded). Copy the template and set strong, unique values first — there is no default password; the one-shot setup container refuses to start if ELASTIC_PASSWORD is blank.

# One-time: create your .env from the template and set strong values
cp scripts/setup/.env.example scripts/setup/.env
# edit .env: ELASTIC_PASSWORD, KIBANA_PASSWORD, LOGSTASH_PASSWORD, KIBANA_ENCRYPTION_KEY (32+ chars)

# Start ELK (reads scripts/setup/.env)
cd scripts/setup && docker compose up -d

3. Accessing Kibana

Because the pipeline is now secured, you will be prompted for a login when you visit Kibana.

  • URL: http://localhost:5601
  • Username: elastic
  • Password: the ELASTIC_PASSWORD value you set in scripts/setup/.env

4. CLI API Access

Any `curl` commands used for interacting with the Elasticsearch backend must now include the `-u` authentication flag. The stack is TLS-secured, so target https:// and pass the CA (or source the password from your shell rather than typing it inline).

# load secrets, then use $ELASTIC_PASSWORD — never paste the literal password
set -a; . scripts/setup/.env; set +a
curl --cacert scripts/setup/certs/ca/ca.crt -u "elastic:$ELASTIC_PASSWORD" https://localhost:9200/_cluster/health

5. Pipeline Log Ingestion

The main setup script has been updated. Logs are securely processed in parallel.

# Interactive Wizard
./scripts/setup/soc_pipeline.sh

# Run PCAP analysis in parallel
./scripts/setup/zeek_run_pcap.sh
