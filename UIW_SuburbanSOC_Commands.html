Suburban-SOC: Command Syntax & Security Update Guide

1. Overview

Following the recent security overhaul of the Suburban-SOC infrastructure, Elasticsearch authentication (X-Pack Security) has been strictly enforced. As a result, starting the ELK pipeline now requires passing a secure password via environment variables to ensure the container boot process seamlessly provisions credentials.

2. Starting the Secured ELK Stack

Use the following command to start the pipeline. If you do not specify a password, it will default to SuburbanSOC2026!.

# Start ELK with the default secure password
docker-compose up -d

# OR Start ELK with a custom secure password
ELASTIC_PASSWORD="YourCustomPassword!" docker-compose up -d

3. Accessing Kibana

Because the pipeline is now secured, you will be prompted for a login when you visit Kibana.

  • URL: http://localhost:5601
  • Username: elastic
  • Password: SuburbanSOC2026! (or the custom password you set)

4. CLI API Access

Any `curl` commands used for interacting with the Elasticsearch backend must now include the `-u` authentication flag.

curl -u elastic:SuburbanSOC2026! http://localhost:9200/_cluster/health

5. Pipeline Log Ingestion

The main setup script has been updated. Logs are securely processed in parallel.

# Interactive Wizard
./scripts/setup/soc_pipeline.sh

# Run PCAP analysis in parallel
./scripts/setup/zeek_run_pcap.sh
